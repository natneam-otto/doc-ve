Order API

The Order API allows you to create fully integrated Orders on behalf of a Clinic connected to your account.

Orders can contain multiple products know as order line items. Order line items are batched by supplier id and sent out as purchase orders to suppliers. When requesting an order status you will recieve the status of each order line item and the purchase order id of each order line item.

Order Status

Check the status of a successful Order object.

Parameters

required
string
Location's 64-character API key

required

ID of the Order for status check
Success Response
patient_id
integer
Associated patient ID with the order, if applicable; otherwise, it will be 0.
promotions
array

An array of objects containing any promotions which apply to the given order. Each array object contains the name of the promotion and the supplier_id

Examples: [{name:"Essentials",supplier_id:4}], This would mean the B&L Essentials promotion applies to this order

boolean

date
The backorder date.
backorder_date
date
The backorder date.
product_id
id
The product ordered on the given order line.
prescription_id
id
The prescription id of the given order line.
status
string
  • Awaiting-Processing : Order has been created and is waiting to completed withing the Otto dashboard
  • Awaiting-Batch : Order has been created and is waiting to be sent out in the next batch order
  • Processing : Purchase Order has been transmitted to the supplier and will be confirmed shortly.
  • Confirmed : Purchase Order has been procured by the supplier and will be fulfilled shortly.
  • Backorder : One or more products on the order are on backorder
  • Fulfilled : Products have been shipped
  • Error : Order has enter an error state
